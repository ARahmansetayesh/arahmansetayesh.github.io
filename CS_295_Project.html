00

Overview

We fine-tune OpenVLA-7B to emit each control command as a named decimal string — dx=0.012 … grip=-1 — over the full 7-DoF action space, and show the representation can be made invariant to the order its dimensions are written in.

Vision-language-action models like OpenVLA usually encode robot actions by repurposing rare vocabulary tokens as discrete bins. We instead represent each action directly as readable text in the model’s native vocabulary, then study a property that only this representation makes possible: invariance to the ordering of the action dimensions.

Mapping controls to arbitrary bin tokens misaligns robot fine-tuning with a VLM’s pretraining distribution — the shared motivation behind VLM2VLA, LAP, and our work. VLM2VLA shows that language-shaped supervision lets LoRA adapt a VLM without destroying VQA; LAP shows the same representation can scale to foundation-model pre-training and embodiment transfer when paired with efficient execution.

The idea of representing actions as language was introduced by VLM2VLA (Hancock et al., 2025), which used positional, translation-only Python lists. Our contribution is a different instantiation built for continuous delta-pose control, plus a controlled study of its behaviour:

  • Named key=value syntax. Each dimension carries a semantic label drawn from the model’s pretraining vocabulary, rather than relying on position to convey meaning.
  • Full 7-DoF including rotation. Prior language-action work handled translation only; our format encodes roll, pitch, and yaw alongside translation and gripper.
  • Order-invariant training. By permuting the dimension order during fine-tuning, we force the model to bind values to labels — a property positional formats cannot express.
01

Method

We fine-tune OpenVLA-7B with 4-bit QLoRA (rank 32, 10 000 steps) on LIBERO-Spatial under three text-action token schemes. All three runs share the same base checkpoint, dataset, and training budget; they differ only in how each 7-DoF command is written as text.

I

Fixed dimension order

The reference encoding: each action is a sequence of named decimal strings in a canonical order. Values are written to fixed precision and recovered losslessly by a regex parser — about 14× finer than OpenVLA’s 256 discrete bins.

Example action string fixed order
dx=0.012 dy=-0.005 dz=0.034 droll=0.001 dpitch=0.002 dyaw=0.005 grip=-1
II

Permuted dimension order

Same decimal strings, but the key=value pairs appear in a fresh random order at every training step. Label masking is anchored per sample so it stays correct under variable token length. The model must bind each value to its label rather than its position.

Same action · re-ordered every beat training-time shuffle
III

Naive quantization

Instead of raw decimals, each motion axis is snapped to a data-driven set of quantization levels (odd counts such as 7, 9, or 11, fitted on LIBERO-Spatial expert actions) and written as controlled vocabulary tokens: zero, or positive/negative plus a level index q1…q11. The gripper is written as open or close.

Example action string 11 quantization levels
x positive q3, y zero, z negative q1, roll zero, pitch positive q2, yaw zero, gripper open

Evaluation runs autoregressive generation on LIBERO-Spatial (10 tasks), parsing each generated string back into a 7-DoF command before stepping the simulator. Cases I and II are reported under Results; Case III is reported under Quantized text-actions.

Results

We evaluate the three text-action schemes on LIBERO-Spatial and LIBERO-PRO. Cases I and II (fixed and permuted decimal strings) appear in the first three subsections below; Case III (naive quantization) in the last.

The headline finding is that permuted training reaches 70% — within striking distance of the canonical model and the bf16 ceiling — despite the model never being shown a consistent dimension order. Learning the task at all under random orderings is non-obvious, and confirms the model relies on labels rather than position.

Under greedy decoding the permuted model produces 19 distinct orderings across 106 steps with a soft preference for certain leading dimensions — an artifact of greedy decoding amplifying small logit gaps. Under temperature-1.0 sampling the distribution is revealed to be near-uniform: 75 distinct orderings in 77 steps, almost never repeating. Every one parses to a valid action.

The model holds two epistemic stances in one output: genuinely uncertain about which dimension to name next (flat distribution), yet confident about each value (sharply peaked). Sampling scrambles the order but not the numbers — which is exactly why task performance can survive it.

Robustness (LIBERO-PRO)

LIBERO-PRO (Zhou et al., 2025) re-runs each LIBERO-Spatial task under controlled perturbations the policy never saw in training, to separate genuine task understanding from trajectory memorisation. We evaluate three of its perturbation axes across all ten spatial tasks, applied independently: position — the initial regions of objects are exchanged, with the instruction, target and goal held fixed; semantic — the instruction is replaced by a paraphrase, with the physical scene unchanged; and task — the target is switched to the other black bowl in the same scene, with the goal predicate updated to match. These runs use the released LIBERO-PRO assets under our own reproducible evaluation protocol; they are not exact reproductions of the paper’s reported numbers (see note below).

The benchmark’s central finding is that VLA models scoring above 90% on vanilla LIBERO can collapse sharply once these perturbations are applied — evidence that high in-distribution scores often reflect memorised layouts and trajectories rather than grounded perception. Our question is narrower: does naming each action dimension, and training the model to bind values to those names regardless of order, change how the policy degrades under perturbation relative to a positional encoding? Position stresses memorised scene geometry, semantic stresses language grounding, and task stresses whether a new goal can be executed in a familiar scene — compositionally the hardest axis for current VLAs. The T=1.0 column repeats the permuted evaluation under temperature-1.0 sampling rather than greedy decoding, testing whether order-invariant sampling — the dimension order scrambled at every step — holds up under perturbation as well as the deterministic greedy policy.

Quantized text-actions

Case III replaces raw decimals with naive quantization. Each non-zero axis is written as a sign (positive/negative/zero) plus a quantization level q1…qN (with odd N swept over 7, 9, and 11); the gripper is open/close.

Not a fine-tuned policy. The results below are an oracle replay ceiling: we take LIBERO-Spatial expert demonstrations, quantize each action through the naive codec, and roll the quantized commands out in sim. No VLA is involved — the question is only what happens if we quantize expert actions? Any learned policy can at best approach these numbers; it cannot exceed them without better actions than the expert.

03

Comparison to Previous Work

Our project sits in the same line of work as recent papers that replace OpenVLA-style bin tokens with readable text. We build directly on the actions-as-language idea, but target a different base model, action grammar, and set of scientific questions.

Hancock et al., 2025 · arXiv:2509.22195

VLM2VLA

Actions as Language fine-tunes a VLM with LoRA by rewriting teleop trajectories into natural language. Low-level actions are nested text lists over translation only; inference is a three-stage hierarchy (subtask → motion plan → action chunk). The main goal is to avoid catastrophic forgetting and preserve VQA while adding control on Bridge V2.

Zha et al., 2026 · arXiv:2602.10556

LAP

Language-Action Pre-Training scales the same core idea to large multi-embodiment pre-training. Actions become templated clauses (move … cm; tilt … deg; …) parsed automatically from continuous control. LAP-3B pairs a language-action VLM with a diffusion action expert for fast execution and reports strong zero-shot cross-embodiment transfer.

Where we differ. We start from OpenVLA rather than Gemma or a custom LAP stack, and ask a complementary question: once actions are text, can the model bind values to named dimensions rather than to string position? Our dx=0.012 … format is finer-grained and explicitly labeled, full rotation is included, and permuted training makes order invariance measurable (e.g. 75 distinct orderings in 77 sampled steps). Case III adds naive quantization — text-action tokens at discrete quantization levels (q1…qN) distinct from LAP’s centimeter/degree templates or VLM2VLA’s nested lists.

Scope. VLM2VLA and LAP target broader generalization (VQA preservation, cross-embodiment pre-training). Our course project is narrower but deeper on representation design: three controlled encodings on one benchmark. We view our work as an OpenVLA-centric extension of the actions-as-text paradigm, with order invariance and quantization as the novel axes.
